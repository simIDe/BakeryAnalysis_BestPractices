Convert Data Types¶

One of the easiest ways to reduce memory usage is by converting data types. For example, float64 can often be safely downcast to float32, and int64 can often become int32 or even smaller data types.

In [5]:
# create a copy of the dataframe for testing memory usage
df_copy = df.copy()


# 'unit_price' is in string format with the Euro symbol, and comma as
#  a decimal separator
# remove the Euro symbol
df_copy['unit_price'] = df_copy['unit_price'].str.replace(' €', '')
# replace comma with dot
df_copy['unit_price'] = df_copy['unit_price'].str.replace(',', '.')

# Downcasting unit_price column
df_copy['unit_price'] = pd.to_numeric(df_copy['unit_price'], downcast='float')
# Downcasting Quantity column
df_copy['Quantity'] = pd.to_numeric(df_copy['Quantity'], downcast='integer')


# Verifying the new memory usage
optimized_memory = df_copy.memory_usage(deep=True).sum() / 1024 ** 2  # in MB
print(f"Optimized memory usage: {optimized_memory:.2f} MB")

Optimized memory usage: 47.52 MB

Modularisation of memory optimization¶

We can create a function that exemplifies modularization by streamlining the task of memory optimization into a single, reusable component. It can be placed in an external Python file for easy import into various projects, enhancing both maintainability and reusability.

In [7]:
def optimize_memory(df, obj_cols_to_optimize = [], in_place=False):
    """
    Optimizes the memory usage of a DataFrame by downcasting numerical and 
    specified object columns.
    
    Parameters:
        df (pandas.DataFrame): DataFrame to optimize
        obj_cols_to_optimize (list): List of object column names to convert to 'category'
        in_place (bool): Whether to modify the original DataFrame in place
    
    Returns:
        pandas.DataFrame: Optimized DataFrame if in_place is False, otherwise None
    """
    
    if not in_place:
        df = df.copy()
    
    # Downcast integer columns
    int_cols = df.select_dtypes(include=['int']).columns
    df[int_cols] = df[int_cols].apply(pd.to_numeric, downcast='integer')
    
    # Downcast float columns
    float_cols = df.select_dtypes(include=['float']).columns
    df[float_cols] = df[float_cols].apply(pd.to_numeric, downcast='float')
    
    # Downcast specified object columns
    for col in obj_cols_to_optimize:
        
        # Check if column exists and is actually of object dtype
        if col in df.columns and df[col].dtype == 'object':
            # Convert to category
            df[col] = df[col].astype('category')
    
    if in_place:
        return None
    else:
        return df

Further Techniques for Optimizing Memory Usage¶

Here, we have demonstrated some common techniques to optimize memory usage, focusing mainly on data type conversion and column pruning. While these steps might seem trivial for our small bakery dataset, they can make a significant difference in larger projects.

Additional Techniques Worth Considering:¶

  1. Load Only Necessary Rows and Columns: When initially exploring data, consider loading only a subset of rows and only the columns you need. This can be easily done using the nrows and usecols parameters in pandas.read_csv() or similar functions in other data-loading libraries.

    # Example: Load only first 1000 rows and selected columns
 df_sample = pd.read_csv("bakery sales.csv", nrows=1000, usecols=['date', 'time', 'article'])

  2. Data Sampling: For preliminary analyses, sometimes working with a representative sample of your data can be as good as using the entire dataset. This also significantly reduces memory usage.

    # Example: Random sampling of 10% of the dataset
 df_sample = df.sample(frac=0.1)

By employing these and previously discussed methods, you can ensure that your Jupyter Notebooks remain efficient, allowing you to tackle datasets that might otherwise require more substantial computing resources.

Average Revenue per Transaction¶

We'll introduce a new Key Performance Indicator (KPI) called Average Revenue per Transaction calculated as follows:

$ \text{Average Revenue per Transaction} = \frac{\text{Total Sales}} {\text{Total Transactions}} $

This KPI provides a measure of the revenue generated on average from each transaction, offering insights into how effectively sales opportunities are being converted into actual revenue.

In [16]:
# Calculation of the Average Revenue per Transaction metric
total_sales = df['unit_price'].sum()
total_transactions = df['ticket_number'].nunique()
avg_revenue_per_transaction = total_sales / total_transactions

print(f"The average revenue per transaction is {avg_revenue_per_transaction.round(2)} € ({total_sales.round(2)} € / {total_transactions} transactions)")
